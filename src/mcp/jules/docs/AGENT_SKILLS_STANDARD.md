# Agent Skills Specification (Standard Protocol)

เอกสารฉบับนี้บันทึกมาตรฐานสากลของ **Agent Skills** เพื่อใช้เป็นแนวทางในการพัฒนาและจัดการทักษะของ AI Agent ในโปรเจกต์นี้และโปรเจกต์อื่นๆ

## 1. ปรัชญาของ Agent Skills
- **Modular:** ทักษะต้องแยกจากกันเป็นส่วนๆ เพื่อให้ก๊อปปี้หรือติดตั้งข้ามโปรเจกต์ได้ง่าย
- **Agnostic:** ทักษะต้องออกแบบมาเพื่อให้ Agent หลายค่าย (Gemini, Claude, Cursor) เข้าใจและใช้งานได้
- **Instruction-Driven:** หัวใจคือไฟล์ `SKILL.md` ที่ระบุ "คำสั่ง" และ "ขอบเขต" อย่างชัดเจน

## 2. โครงสร้างไฟล์มาตรฐาน (The SKILL.md Format)
ทุกทักษะต้องมีไฟล์ `SKILL.md` ที่ประกอบด้วย:
- **Metadata (Frontmatter):** ระบุ `name` และ `description`
- **Instructions:** คำแนะนำระดับสูงเพื่อให้ Agent เปลี่ยนพฤติกรรมเมื่อ Activate
- **Constraints:** ข้อจำกัด (เช่น ห้ามเขียนโค้ด, ห้ามลบไฟล์)
- **Tools/Resources:** เครื่องมือที่ทักษะนี้มีให้ใช้งาน

## 3. การจัดการผ่าน Skills CLI (`npx skills`)
มาตรฐานการรันคำสั่งที่ Agent ต้องปฏิบัติตามเพื่อความรวดเร็ว:
- **ค้นหา:** `npx skills find "<query>"`
- **ติดตั้งแบบรวดเร็ว (Headless):** `npx skills add <repo> -g -y`
- **เจาะจง Agent:** `npx skills add <repo> --agent * -y`
- **ตรวจสอบสถานะ:** `npx skills list --json`

## 4. บทเรียนสำคัญสำหรับการพัฒนา
- **ห้ามจำลองทักษะ:** หากทักษะนั้นมีอยู่ในระบบนิเวศ (`skills.sh`) ให้ติดตั้งผ่าน CLI เสมอ ห้ามพยายามเขียนคำแนะนำขึ้นมาเองใหม่
- **Single Source of Truth:** เมื่อติดตั้งเสร็จ ให้ Agent ไปอ่าน `SKILL.md` ที่ติดตั้งจริงในเครื่องทันทีเพื่อดึงบริบทที่ถูกต้องที่สุด
