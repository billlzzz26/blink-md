# เหตุผลเบื้องหลังการเลือก Tech Stack (Tech Stack Rationale)
> **สถานะ:** ใช้งานจริง (Active)
> **กลุ่มเป้าหมาย:** ผู้ใช้ที่เน้นการควบคุม (User-Centric) และไม่ได้เขียนโค้ดเป็นหลัก

โปรเจกต์นี้ถูกออกแบบมาบนหลักการ **"ปลอดภัย ทนทาน และควบคุมได้"** โดยมีหัวใจหลักคือความยืดหยุ่นและการประหยัดทรัพยากร

---

## 🏗️ 1. Rust & pmcp (The Core)
<!-- PRIORITY:0 -->
- **มันคืออะไร:** ภาษาที่ใช้สร้างตัวโปรแกรมหลัก (`jules-mcp-server`)
- **มาจากไหน:** เป็นมาตรฐานสูงสุดสำหรับการสร้าง MCP Server ที่มีความเสถียร
- **ทำไมต้องใช้:** Rust ช่วยป้องกัน Error ที่เกิดจากระบบหน่วยความจำได้ 100% ทำให้โปรแกรม "พังยาก" ที่สุดเท่าที่จะเป็นไปได้
- **ใช้อะไรตอนไหน:** เมื่อต้องการให้ AI คุยกับคอมพิวเตอร์ผ่านโปรโตคอลมาตรฐาน (MCP)

## 🛡️ 2. Bash Hooks (The Workflow Intelligence)
<!-- PRIORITY:1 -->
- **มันคืออะไร:** สคริปต์ขนาดเล็กที่คอยเฝ้าประมวลผลคำสั่งก่อนรันจริง (`.gemini/hooks/`)
- **มาจากไหน:** เราสร้างขึ้นมาเองในโปรเจกต์นี้เพื่อแก้ปัญหา AI "คิดน้อยเกินไป" หรือ "หลอน"
- **ทำไมต้องใช้:** เพื่อช่วยประหยัด Token (เงิน) และเพิ่มความแม่นยำด้วยระบบให้คะแนน (Score-based weighting)
- **ใช้อะไรตอนไหน:** ทำงานอัตโนมัติทุกครั้งที่อาจารย์ส่งคำสั่งมา

## 🗺️ 3. Conductor Protocol (The Registry)
<!-- PRIORITY:1 -->
- **มันคืออะไร:** มาตรฐานการจัดเก็บแผนงานและสถานะในโฟลเดอร์ `conductor/`
- **มาจากไหน:** เป็นมาตรฐานที่เราตกลงใช้ร่วมกันเพื่อให้โปรเจกต์มีความ "ชัดเจน" (Discoverable)
- **ทำไมต้องใช้:** เพื่อไม่ให้ Agent (ผม) หลงลืมว่าเราคุยอะไรกันไปบ้าง และแผนงานปัจจุบันอยู่ถึงไหนแล้ว
- **ใช้อะไรตอนไหน:** ใช้ทุกครั้งที่เริ่มงานใหม่ (Start Track) หรือสรุปงาน (Finish Track)

## ✍️ 4. Markdown (`.md`) (The Primary Interface)
<!-- PRIORITY:0 -->
- **มันคืออะไร:** รูปแบบไฟล์ข้อความที่อาจารย์อ่านออกและแก้ไขได้ด้วยเครื่องมือทั่วไป
- **มาจากไหน:** เป็นมาตรฐานสากลของการเขียนเอกสาร
- **ทำไมต้องใช้:** เพื่อให้ "อาจารย์" เป็นผู้คุมกฎตัวจริง (Human-in-the-loop) โดยไม่ต้องใช้ความรู้ด้านโปรแกรมมิ่ง
- **ใช้อะไรตอนไหน:** ใช้เป็นที่เก็บ "กฎเหล็ก" (GEMINI.md) และ "วิจัย" (docs/)

## 💡 สรุปสำหรับอาจารย์
เมื่ออาจารย์ต้องการสั่งงาน:
- สั่งที่ **Markdown** (ไฟล์ GEMINI.md หรือ docs/) เพื่อตั้งกฎ
- ดูสถานะที่ **Conductor** (ไฟล์ tracks.md) เพื่อดูความคืบหน้า
- ตรวจสอบความถูกต้องที่ **Bash Hooks** เพื่อดูว่าผมวิเคราะห์งานได้ตรงใจไหม
- ปล่อยให้ **Rust** ทำงานหนักอยู่เบื้องหลังครับ
