# มาตรฐานการจัดการความจำ (Interactive Memory Protocol)
<!-- PRIORITY:0 -->
> **สถานะ:** ใช้งานจริง (Active)
> **เวอร์ชัน:** 1.0

เพื่อให้การบันทึกข้อมูลและ "ความจำ" ของ Agent มีความเป็นระเบียบและไม่รบกวน Context Window ของการทำงาน กฎต่อไปนี้คือมาตรฐานที่ต้องปฏิบัติ:

---

## 🛡️ กฎการบันทึกความจำ (Trigger Rules)

เมื่อ Agent ตรวจพบคำสั่งหรือ Intent ที่ต้องการให้ "จำข้อมูล" (เช่น *จำไว้*, *จำไว้นะ*, *ช่วยจำไว้*, *โปรดจำไว้*, *จำข้อมูลนี้*, หรือ *remember this*) ห้ามเรียกใช้ Tool บันทึกความจำโดยตรง **ต้องผ่านกระบวนการยืนยันดังนี้**:

### 1. การเรียกใช้ `ask_user` (Mandatory)
Agent ต้องถามผู้ใช้เสมอว่าต้องการบันทึกข้อมูลในระดับใด:
- **Global Memory:** บันทึกในคลังความรู้กลาง (ใช้ได้ทุกโปรเจกต์)
- **Project Memory:** บันทึกเฉพาะในโปรเจกต์ปัจจุบัน (Project-Specific)

### 2. การแยกประเภทความจำ
- **Global:** เก็บไว้ใน `~/.gemini/GEMINI.md` หรือ Global Knowledge Base
- **Project:** เก็บไว้ใน `./GEMINI.md`, `./docs/` หรือ Conductor Tracks

---

## 🧹 การจัดการความจำในระยะยาว (Archiving)

เพื่อป้องกันปัญหา **Context Overflow** (ความจำเยอะจน Agent หลอน):
- จะมีระบบ **Memory Manager (Cron Job)** ทำหน้าที่สรุป (Summarize) และคัดกรอง (Prune) ข้อมูลเก่า
- ความจำที่ "จบงานแล้ว" (Completed Tasks) จะถูกย้ายไปที่ Archive ทุก 30 วัน

---

## 💡 แนวทางการเขียนความจำ (Formatting)
ความจำที่ดีควรมีลักษณะ:
- **High Signal:** สรุปเฉพาะแก่นสำคัญ ไม่เวิ่นเว้อ
- **Contextual:** ระบุเหตุผลว่าทำไมถึงต้องจำ (เช่น "ใช้เป็นกฎเหล็กเมื่อทำงานกับผู้ใช้")
- **Traceable:** มีที่มาของความจำ (เช่น "จากบทเรียนในเซสชัน X")
